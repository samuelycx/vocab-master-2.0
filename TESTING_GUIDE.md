# P0优化测试指南

## 📋 测试前准备

### 1. 环境要求
- Node.js v16+
- 微信开发者工具
- 微信小程序AppID

### 2. 安装依赖
```bash
# 进入client-uni目录
cd client-uni

# 安装依赖
npm install
```

### 3. 云开发配置
1. 在微信开发者工具中打开项目
2. 开通云开发环境
3. 创建以下数据库集合：
   - `users` (用户表)
   - `pk_rooms` (PK房间表)
   - `study_records` (学习记录表)
   - `words` (词汇表)

---

## 🧪 测试清单

### P0-1: 安全优化测试

#### 1. 设备绑定测试
**测试步骤**：
1. 首次打开小程序，自动登录
2. 查看控制台，确认设备信息已传递
3. 在数据库`users`表中查看`deviceId`字段是否已保存
4. 尝试在另一台设备登录同一账号
5. 应该看到提示："此账号已在其他设备绑定，如需更换设备请联系家长"

**预期结果**：
- ✅ 首次登录自动绑定设备
- ✅ 设备信息保存在数据库
- ✅ 其他设备无法登录

#### 2. 管理后台移除测试
**测试步骤**：
1. 检查底部导航栏，确认没有管理后台入口
2. 尝试通过代码跳转到`admin-dashboard`
3. 应该无法访问

**预期结果**：
- ✅ 管理后台入口已隐藏
- ✅ 无法通过任何方式访问管理后台

#### 3. WebSocket改造测试
**测试步骤**：
1. 进入PK竞技页面
2. 点击"开始匹配"
3. 查看控制台，确认没有WebSocket连接错误
4. 查看数据库`pk_rooms`表，确认房间已创建
5. 等待匹配（可以开两个模拟器测试）

**预期结果**：
- ✅ PK匹配功能正常
- ✅ 使用云开发实时数据库
- ✅ 没有外部WebSocket连接

#### 4. 敏感信息移除测试
**测试步骤**：
1. 检查`.env`文件，确认敏感信息已注释
2. 检查`.gitignore`，确认`.env`已被忽略
3. 尝试提交代码，确认`.env`不会被提交

**预期结果**：
- ✅ 敏感信息已移除
- ✅ 不会被提交到代码库

## 🐛 常见问题排查

### 1. 登录失败
**可能原因**：
- 云开发环境未开通
- 数据库集合未创建

**解决方案**：
```bash
# 检查云开发环境
# 在微信开发者工具中：云开发 -> 设置 -> 环境ID

# 创建数据库集合
# 在云开发控制台手动创建
```

### 2. PK匹配失败
**可能原因**：
- `pk_rooms`集合未创建
- 数据库权限设置错误

**解决方案**：
```bash
# 在云开发控制台创建pk_rooms集合
# 设置权限为：所有用户可读写
```

### 3. UI显示异常
**可能原因**：
- 样式未生效
- 缓存问题

**解决方案**：
```bash
# 清除缓存
# 在微信开发者工具中：工具 -> 清缓存 -> 清除全部

# 重新编译
npm run dev:mp-weixin
```

### 4. 动画不流畅
**可能原因**：
- 设备性能问题
- 动画代码优化不足

**解决方案**：
```bash
# 在真机上测试
# 使用微信开发者工具的"真机调试"功能
```
