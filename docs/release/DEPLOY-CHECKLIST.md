# 部署与同步防回归清单

本清单用于避免“同步/部署引入非目标改动”的问题。每次同步或发布必须按顺序执行。

## 1) 变更范围锁定（必须写清）
- 明确本次只动哪些目录（例如：`web/` 或 `server/` 或 `miniprogram/`）。
- 未经确认，禁止跨目录改动。

## 2) 同步前变更摘要确认
- 在同步/推送前给出：**目录 + 文件数 + 主要改动**。
- 必须得到确认后再执行同步或推送。

## 3) 构建产物一致性
- 只用本地构建产物同步到服务器（例如：`web/dist`）。
- 同步前后记录构建时间与产物大小，避免“旧产物覆盖新代码”。

## 4) 固定发布顺序（不可跳步）
1. 本地改动完成
2. 本地测试（能跑就跑）
3. 本地构建（生成 `dist`）
4. 同步到服务器
5. 重启服务
6. 访问验证

## 5) 禁止结构性合并混入日常同步
- 结构调整/重构必须独立提交、独立发布。
- 日常同步禁止合并历史工作区或大范围结构改动。

## 6) 明确“不动某模块”的锁定口令
- 你说“本次只动 X”，我将只在该目录内改动。
- 如需扩大范围，必须先确认。
