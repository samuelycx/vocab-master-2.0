# Vocab Master 2.0 UI/UX 升级方案

## 零、微信小程序安全区域 (Safe Area)

微信小程序顶部有状态栏（Status Bar）和导航栏，底部有安全区域：

```
┌─────────────────────────────┐
│  ← 状态栏 (~44rpx)          │  ← 需要 padding-top: var(--status-bar-height)
├─────────────────────────────┤
│  ← 自定义导航栏 (~44rpx)     │
├─────────────────────────────┤
│                             │
│        页面内容               │
│                             │
│                             │
│                             │
├─────────────────────────────┤
│  ← 底部安全区 (~34rpx)       │  ← 需要 padding-bottom: var(--safe-area-inset-bottom)
└─────────────────────────────┘
```

### 安全区域CSS变量
```css
/* 在 page 或最外层 view 设置 */
page {
  padding-top: var(--status-bar-height, 0px);
  padding-bottom: var(--safe-area-inset-bottom, 0px);
  min-height: 100vh;
}
```

### 关键原则
- **顶部返回按钮**：距离顶部 ≥ 44rpx + 状态栏高度
- **底部操作按钮**：距离底部 ≥ 34rpx + 安全区域
- **重要内容区域**：上下各预留 48rpx 边距
- **fixed 定位元素**：必须考虑安全区域

## 三、动效规范

### 3.1 过渡动画
```css
/* 快速过渡 - 按钮点击 */
--transition-fast: 0.15s ease;

/* 正常过渡 - 卡片切换 */
--transition-normal: 0.3s ease;

/* 慢速过渡 - 页面切换 */
--transition-slow: 0.5s ease;

/* 弹性过渡 - 弹跳效果 */
--transition-bounce: 0.5s cubic-bezier(0.34, 1.56, 0.64, 1);
```

### 3.2 动画类型

**1. 弹入 (Pop In)**
```css
@keyframes popIn {
  0% { opacity: 0; transform: scale(0.5); }
  100% { opacity: 1; transform: scale(1); }
}
```

**2. 滑入 (Slide Up)**
```css
@keyframes slideUp {
  0% { opacity: 0; transform: translateY(40rpx); }
  100% { opacity: 1; transform: translateY(0); }
}
```

**3. 卡片翻转 (Card Flip)**
```css
@keyframes cardFlip {
  0% { transform: rotateY(0deg); }
  100% { transform: rotateY(180deg); }
}
```

**4. 摇晃 (Shake)**
```css
@keyframes shake {
  0%, 100% { transform: translateX(0); }
  20%, 60% { transform: translateX(-10rpx); }
  40%, 80% { transform: translateX(10rpx); }
}
```

**5. 脉冲 (Pulse)**
```css
@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.05); }
}
```

**6. 漂浮 (Float)**
```css
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-10rpx); }
}
```

### 3.3 动效触发点

| 场景 | 动效 | 时长 |
|------|------|------|
| 页面加载 | 渐入 + 滑入 | 0.5s |
| 按钮点击 | 缩小 0.95 + 回弹 | 0.15s |
| 卡片悬停 | 上浮 4rpx + 阴影加深 | 0.2s |
| 答题正确 | 绿色背景 + 缩放 | 0.3s |
| 答题错误 | 红色背景 + 摇晃 | 0.3s |
| 卡片翻转 | 3D旋转 180° | 0.6s |
| 进度更新 | 宽度平滑过渡 | 0.5s |
| 结果弹窗 | 放大弹入 | 0.4s |
| 学习完成 | 撒花特效 | 2s |

## 五、技术实现建议

### 5.1 微信小程序动画方案
1. **CSS动画** - 简单过渡效果 (推荐)
2. **wx.createAnimation** - 复杂序列动画
3. **Lottie** - 复杂特效动画
4. **Canvas** - 粒子特效 (撒花等)

### 5.2 性能注意事项
- 避免频繁触发重排
- 合理使用 `transform` 和 `opacity`
- 动画元素使用 `will-change`
- 复杂特效使用 `requestAnimationFrame`
