# 成就墙图标与展示规则设计（Achievement Wall V2）

日期：2026-03-12

## 目标
在 `pencil-new.pen` 的 `AchievementWall Concept V2 (v0R7V)` 中完整展示所有成就条目，并与业务逻辑保持一致：首次使用自动解锁 `growth_first_use`（初来乍到），其余全部未解锁（灰度）作为初始默认状态。

## 用户/产品约束
- 初始状态为“仅初来乍到解锁，其余未解锁”。
- 所有成就必须全部展示（6 个分类：成长/毅力/技巧/博学/财富/特殊）。
- 每个成就需要独立图标，不允许复用图标。
- 视觉风格参考 `Y00wj / fAX05 / jeprl / DaYD6`（具象、平面、童趣、可辨识）。
- 未解锁状态需有清晰的灰度/低对比表现。

## 方案选择
采用 **方案 A**：初始态仅“初来乍到”高亮，其余全部灰度；顶部“已解锁 1 / 31”与真实逻辑一致。

## 设计规范（Pencil）
1. **分类完整**：每个分类卡片内包含该分类全部成就（按后台 ID 顺序展示）。
2. **解锁态**：
   - `growth_first_use` 显示为解锁态（彩色图标 + 正常文字）。
   - 其余成就为未解锁态（图标灰度+透明度降低，文字浅灰）。
3. **网格布局**：每个分类卡内以 2 列网格排布，允许多行自动换行。
4. **图标来源**：全部从 `achievement wall icon` 图标库中取用（每个成就独立）。

## 数据与逻辑对齐
- 成就定义以云函数 `progress` 的 `DEFAULT_ACHIEVEMENTS` 为准。
- UI 初始态与“首次使用自动授予”逻辑一致：默认 1 解锁。

## 验收标准
- v0R7V 页面完整展示 31 个成就（6 分类全覆盖）。
- 仅 `growth_first_use` 解锁态，其余灰度。
- 视觉与 Pencil 样式一致，无重叠、对齐一致。
