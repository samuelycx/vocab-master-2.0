# 词库乱码审计与规则沉淀设计

## 背景与目标
我们需要基于**服务器词库**做一次全量扫描，输出：
1) 乱码判定规则（可复用标准）
2) 异常清单（便于修复与后续导入校验）

## 数据源
- 服务器 SQLite：`/var/www/vocab-master/server/prisma/dev.db`
- 表：`Word`（字段：`text`, `meanings`, `examples`, `phonetic`, `partOfSpeech` 等）

## 非目标
- 不在本次任务中修复词库数据
- 不改写词库格式或导入流程
- 不改业务逻辑

## 方案概述（已确认）
**方案 A：复制服务器 DB 到本地扫描**
- 通过 SSH 只读复制 `dev.db` 到本地临时目录
- 本地脚本解析并输出审计文档

## 规则沉淀方向（初始规则）
扫描时根据实际数据补充/调整，最终输出在文档中：
1. 替换字符/乱码符：包含 `�` / `\uFFFD`
2. JSON 解析失败：`meanings` / `examples` 非法 JSON
3. 控制字符：出现 `\u0000`–`\u001F`（保留换行/制表除外）
4. 异常字符占比：非中英文/常见标点占比超过阈值（初始阈值 35%）
5. 常见错码片段：如 `Ã©`、`â€™`、`â€œ` 等
6. 过长连续异常字符（如 6+ 连续无意义符号）

## 输出文档结构
输出文档：`docs/release/WORDDATA_ENCODING_AUDIT.md`

包含：
- 扫描时间与数据源信息
- 规则列表（含解释与示例）
- 异常汇总（计数）
- 异常清单（表格：wordId / text / 字段 / 触发规则 / 片段）

## 实现步骤
1. 服务器复制数据库到本地临时目录
2. 扫描脚本：
   - 读取 SQLite
   - 逐行检查规则
   - 记录命中规则及片段
3. 生成 MD 文档
4. 输出异常统计 + 清单

## 风险与控制
- 数据泄露风险：本地文件只保留到输出完成，完成后删除
- 扫描耗时：脚本仅扫描 `Word` 表，控制在 5–10 分钟

## 验证标准
- 输出文档完整生成
- 规则条目清晰可复用
- 异常清单包含字段定位与原因
