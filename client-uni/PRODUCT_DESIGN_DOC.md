# Vocab Master 2.0 - 产品设计文档

## 1. 产品概述

**产品名称**: Vocab Master 2.0（词汇大师）  
**产品定位**: 游戏化词汇学习小程序  
**目标用户**: 中小学生、托福/GRE考生、英语爱好者  
**核心卖点**: 通过PK竞技、成就系统、每日挑战等游戏化机制提升词汇学习效率

## 2. 功能模块

### 2.1 用户系统
- **自动登录**: 基于微信openid，无需注册
- **用户档案**: 
  - 用户名（自动生成 player_xxxx 或自定义）
  - 等级系统（Level 1-∞，每级需要100 XP）
  - 称号系统（根据等级自动解锁）
  - 头像（emoji选择）
  - 连续打卡天数（Streak）
  - 金币余额（Coins）

### 2.2 单词数据
**单词数据字段**:
- text: 单词文本
- partOfSpeech: 词性
- meanings: 释义（JSON数组）
- examples: 例句（JSON数组）
- audio: 发音（有道TTS API）

### 2.3 学习模式

#### A. 日常学习模式
- **界面**: 单词卡片 + 4选1答题
- **流程**:
  1. 展示单词（显示单词文本、词性）
  2. 用户从4个释义选项中选择正确答案
  3. 即时反馈（正确/错误）
  4. 自动播放发音（可选）
  5. 答对后进入下一题
- **计分**:
  - 基础XP: 10分/题
  - 连击加成: 每5题连击+10 XP
  - 金币: 1+ floor(连击/10) 枚/题

#### B. 复习模式
- **触发条件**: 用户有已学习但需复习的单词
- **算法**: SM-2间隔重复算法
- **状态**: 
  - MASTERED（已掌握）
  - LEARNING（学习中）
  - 新词

#### C. 错题本模式
- 展示用户答错过的单词
- 针对性练习

#### D. PK竞技场模式
**子模式**:
1. **人机对战**: 与AI Bot对战
2. **在线匹配**: 实时匹配其他玩家（需WebSocket服务器）
3. **私人房间**: 创建/加入房间（开发中）

**PK机制**:
- 每题限时15秒
- 答对得分: 10 + floor(连击/2) 分
- Bot AI: 3秒间隔，80%正确率
- 胜利条件: 先达到200分
- 奖励: 获胜方获得双倍XP和金币

### 2.4 成就系统
**成就类型**:
- 连续学习天数（Streak）
- 累计学习单词数
- PK胜利次数
- 等级达成

**展示形式**: 
- 成就墙（AchievementWall）
- 解锁弹窗动画
- 徽章展示

### 2.5 社交系统
**功能**:
- 排行榜（Leaderboard）
- 关注/粉丝系统
- 动态流（Feed）
- 好友PK挑战

### 2.6 词汇量统计
**数据展示**:
- 总学习单词数
- 已掌握单词数
- 学习中单词数
- 搜索已学单词

### 2.7 设置系统
**选项**:
- 音效开关
- 主题切换（学术风/活泼风）
- 暗黑模式

## 3. 用户流程

### 3.1 首次使用流程
```
微信授权 → 自动生成用户 → 显示欢迎页 → 进入首页Dashboard
```

### 3.2 日常学习流程
```
Dashboard → 选择"开始学习" → 答题界面 → 完成/退出 → 结果页 → Dashboard
```

### 3.3 PK流程
```
Dashboard → 进入PK竞技场 → 选择模式 → 匹配中 → 对战界面 → 结果页 → Dashboard
```

## 4. 技术架构

### 4.1 前端
- **框架**: uni-app + Vue 3
- **状态管理**: reactive (Vue 3 Composition API)
- **样式**: TailwindCSS + 自定义CSS变量
- **构建**: Vite

### 4.2 后端
- **平台**: 微信云开发（TCB）
- **云函数**: 
  - auth（用户认证）
  - words（单词查询）
  - progress（学习进度）
  - admin（管理后台）
- **数据库**: 
  - users（用户表）
  - words（单词表）
  - progress（学习记录表）

### 4.3 实时通信
- WebSocket（Socket.io）
- 用于PK实时对战

## 5. 现有UI问题（供设计师参考）

### 5.1 已知问题
- 样式过于复杂，加载性能受影响
- 部分动画效果在微信小程序中不流畅
- 深色模式切换有闪烁
- 响应式布局在某些屏幕尺寸下显示异常

### 5.2 设计风格建议
- **当前主题**: 
  - 学术风: 绿色系，简洁专业
  - 活泼风: 橙色系，年轻活力
- **建议改进方向**:
  - 简化视觉层级
  - 优化加载速度
  - 统一组件风格
  - 增强游戏化视觉反馈

## 7. 关键界面说明

### 7.1 首页Dashboard
**功能入口**:
- 开始学习（大按钮，突出显示）
- 继续旅程（显示进度）
- PK竞技场
- 我的词汇量
- 设置

**信息展示**:
- 用户等级、XP进度条
- 连续打卡天数
- 今日已学单词数
- 待复习单词数

### 7.2 答题界面
**元素**:
- 顶部：进度条、计时器、连击数
- 中部：单词（大号显示）、词性
- 底部：4个选项按钮
- 反馈：正确/错误动画、得分飘字

### 7.3 PK界面
**元素**:
- 顶部：双方头像、分数对比
- 中部：倒计时、单词
- 底部：选项按钮
- 特效：得分动画、胜利/失败特效

## 8. 游戏化机制

### 8.1 经验值系统
- 学习获得XP
- 等级提升解锁称号
- 称号展示在用户卡片

### 8.2 金币系统
- 答题获得金币
- 可用于购买皮肤/道具（未来功能）

### 8.3 连击系统
- 连续答对增加连击数
- 连击越高，得分倍数越高
- 断连有提示

### 8.4 每日挑战
- 每日目标：学习N个单词
- 完成奖励：额外金币+XP
- 连续打卡奖励递增
